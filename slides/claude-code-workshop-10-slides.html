Essential Keyboard Shortcuts

  • Ctrl+C - Cancel current operation
  • Ctrl+D - Exit Claude session
  • Escape - Interrupt Claude mid-action
  • Double Escape - Edit previous prompt
Permission Strategy: Start conservative, add permissions as you build trust and understanding

CLAUDE.md - Your Project Memory

File Hierarchy (Priority Order)

  • ./CLAUDE.md - Project root (team shared)
  • ./subdir/CLAUDE.md - Directory specific
  • ~/.claude/CLAUDE.md - Global user settings
# Essential CLAUDE.md Content ## Build Commands - npm run dev: Start development server - npm test: Run tests - npm run lint: Must pass before commits ## Code Style - Use TypeScript for all new files - Follow existing naming conventions - Add JSDoc comments for public functions ## Architecture @./docs/architecture.md # Import other files
Pro Tip: Keep CLAUDE.md concise and actionable. It becomes part of Claude's context automatically.
Speaker Notes: CLAUDE.md is your project's memory system. The hierarchy means more specific files override general ones. The @ syntax lets you import other files. Keep the main file focused on essential daily info. The architecture import example shows how to modularize larger documentation.

Permission Management & Security

Permission Hierarchy (Highest to Lowest)

  • Enterprise policies (managed by IT)
  • Command line arguments
  • Local project settings
  • User settings
# Essential Permission Commands /permissions add Edit # Allow file editing /permissions add Bash(git:*) # Git operations /permissions add Bash(npm:test) # Testing only /permissions list # View current permissions # CLI override (session only) claude --allowedTools Edit,Bash
‚úÖ Safe Permissions:
  • Edit, Read
  • Bash(git:*)
  • Bash(npm:test)
  • Bash(npm:lint)
‚ùå Avoid in Production:
  • Bash(*) - Too broad
  • Bash(rm:*) - Dangerous
  • Bash(curl:*) - Network access
Speaker Notes: Security is built into Claude Code through the permission system. Start with safe permissions and add more as needed. Enterprise policies can enforce security standards. The CLI override is useful for scripting. Never use overly broad permissions like Bash(*) in production environments.

Performance & Context Optimization

Context Window Management

  • Use /clear frequently between tasks
  • Split large tasks into focused sessions
  • Use multiple Claude instances for complex work
  • Monitor for "context window" warnings

Optimized CLAUDE.md Structure

# CRITICAL INFO (always needed) - Build command: npm run build - Test command: npm test - Linting: npm run lint (MUST pass) # IMPORTS (pull in as needed) @./docs/architecture.md @./standards/coding-style.md # CONTEXT-SPECIFIC (loaded on demand) @./frontend/patterns.md @./backend/database-patterns.md

Performance Anti-patterns

  • Massive single CLAUDE.md files (200+ lines)
  • Working in one session for hours
  • Including rarely-needed documentation
  • Not using /clear between unrelated tasks
Speaker Notes: Performance is about managing Claude's context window effectively. Use /clear liberally - it's like giving Claude a fresh start. Structure your CLAUDE.md with priority - critical info first, then imports for detailed info. The anti-patterns section shows what kills performance.

Troubleshooting & Recovery

Common Issues & Solutions

Context Issues:

  • Context full: Use /clear
  • Slow responses: Check connection
  • Wrong context: Start fresh session

Permission Issues:

  • Check /permissions list
  • Add needed permissions
  • Use CLI override for testing

Recovery Strategies

  1. Interrupt: Press Escape to stop
  2. Redirect: "Try a different approach using..."
  3. Undo: "Undo the last changes"
  4. Reset: git checkout -- filename

Debug Commands

claude --verbose # Detailed logging claude --mcp-debug # MCP troubleshooting time claude -p "..." # Performance timing
Speaker Notes: Everyone will hit issues - this slide gives you the tools to recover quickly. Escape is your friend for interrupting Claude. The recovery strategies are in order of preference. Debug commands help diagnose issues. Remember: mistakes are learning opportunities for both you and Claude.
